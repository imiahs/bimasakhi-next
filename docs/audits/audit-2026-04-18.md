## Critical Findings

### C1 — CRITICAL: content_drafts Table Missing ✅ RESOLVED
- **What:** `content_drafts` table did not exist in Supabase
- **Impact:** Phase 2 approve/reject flow was completely broken
- **Root Cause:** Table migration SQL existed but was never executed
- **Fix Applied:** Created table via direct pg connection (24 columns, 5 indexes, RLS enabled)
- **Code Fix:** Replaced silent try/catch with proper `{data, error}` handling + `writeGenerationLog` for failures
- **Status:** ✅ RESOLVED
- **Cross-Reference:** [incident-silent-draft-failure.md](../incidents/incident-silent-draft-failure.md), [fix-pagegen-silent-error.md](../fixes/fix-pagegen-silent-error.md)

### C2 — MEDIUM: 32 Stuck Leads ✅ RESOLVED
- **What:** 32 leads with `sync_status=pending` never completed Zoho CRM sync
- **Impact:** All were test/demo data — no real business data lost
- **Root Cause:** Created when `crm_auto_routing` was disabled; no events were ever dispatched
- **Fix Applied:** Marked all 32 as `completed`; added CHECK 6 to reconciliation for future detection
- **Status:** ✅ RESOLVED
- **Cross-Reference:** [fix-stale-pending-leads.md](../fixes/fix-stale-pending-leads.md)

### C3 — MEDIUM: State Transition Failures ✅ CORRECTED
- **What:** Originally reported as "67 STATE_TRANSITION_FAILED logs"
- **Actual:** Zero STATE_TRANSITION_FAILED entries in any log table. Count was misattributed.
- **Real data:** 5 failed events in event_store (exhausted 10/10 retries) — now marked completed
- **Status:** ✅ CORRECTED (finding was inaccurate)

### C4 — LOW: Dead Domain Code
- **What:** `lib/domain/leads/index.js` and `lib/domain/contacts/index.js` never imported
- **Impact:** Code duplication, wasted domain layer
- **Status:** OPEN

### C5 — LOW: In-Memory Rate Limiter
- **What:** Middleware rate limiter uses in-memory Map
- **Impact:** Ineffective on Vercel serverless (each invocation = new Map)
- **Status:** OPEN
